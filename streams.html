Streams with Promises

Abstract

This document proposes integration of DOM Promises and Streams API.

Streams Model

A stream represents a data sources for which the total size is unknown until EOF is reached. Using Promises, Streams provides read once access model. Data queued to a stream is read in FIFO order. Data pushed to a stream can be read out only once from it.

Streams API

interface Stream {
  AbortableProgressPromise<ArrayBuffer> read(optional unsigned long long size);
};

A stream's read(maxSize) method takes optionally a maxSize, and runs these steps:

  1. If has been neutered or the read posision is -1, throw an InvalidStateError exception and terminate this algorithm.
  2. Let read promise be a new abortable progress promise of the array buffer and read resolver be read promise's associated resolver.
  3. Let pos be the stream's read position.
  4. If called with the optional maxSize, set the stream's read position to pos + maxSize.
  5. Otherwise, set the stream's read position to -1.
  6. Append read resolver to pending read resolvers.
  7. Return the read() method with promise, but continue to process the steps in this algorithm.
  8. Read data from pos up to maxSize bytes or all data from pos if maxSize is not specified.
  9. As data from the stream becomes available, do the following,
    1. If readType is the empty string or "text"

      Let result be the result of the following algorithm.

      2. Blah
      If readType is "arraybuffer"

      Let result be an array buffer created from the read data.

      If readType is "blob"

      Let result be an blob created from the read data.

    2. If any error has occurred during the last step, call reject of read resolver with a DOMError object that indicates the kind of stream error that has occurred, and terminate this algorithm.
    3. User agents should release resource for storing the read data other than result.
    4. Call progress of read resolver with result.
    5. If the end of data is reached or finished reading maxSize bytes, call fulfill of read resolver and terminate this algorithm.
